**GOODLAND TBS SHERMAN (`loc 12001`), gas day 2026‑06‑15, Timely, Delivery**.

**Origin legend**
- **scrape ← `COL`** — taken from that column of the site's download (1:1).
- **scrape ← `COL` (→/+)** — derived from a scraped column (`GAS_DAY` → date and +1 day; `POST_DT` → date).
- **site const** — a fixed value shown on the page / set by the gas-day standard (`9` = 09:00 CCT).
- **site pipeline id** — `302`, the pipeline number from the URL (char(3)).
- **direction radio** — the page's `Receipt`/`Delivery` radio label (records which pull this is).
- **minted** — generated by the scraper (only `scrape_date`, the run timestamp).
- **omitted** — not written at all, so the DB supplies it on INSERT: `flow_uuid` (`gen_random_uuid()`) and `load_date` (`now()`). `flow_uuid` can't be shipped NULL (NOT NULL), and neither is the scraper's to invent.
- **NULL — …** — written but empty, left for Criterion's loader: surrogate keys, internal code-table values, or fields that don't apply to point capacity (storage/no-notice).

Notes:
- `scheduled_quantity ← NET_SCHD_QTY` (not `SCHD_QTY`) so that
  `operating − scheduled = operationally_available` — verified here:
  `13000 − 2055 = 10945`.
- The same `loc 12001` on the **Receipt** pull carries `loc_purp_desc = Receipt Location`
  and its own (different) quantities — direction is the only thing distinguishing them.
